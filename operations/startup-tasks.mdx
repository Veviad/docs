## Contract

<code>STARTUP_TASKS</code> is a comma-separated list of <code>task-name:version-key</code> entries. The database ledger ensures each key completes once across concurrent containers.

Supported tasks are:

- <code>essential-seeders</code>
- <code>search-ensure-settings</code>
- <code>search-reindex</code>
- <code>fresh-db-seed</code>

## Blocking behavior

Essential seeders, search settings, and fresh database seed block startup on failure. <code>search-reindex</code> is non-blocking: another container may already own it, and a failed reindex is recorded without preventing the application from starting.

## Production example

```dotenv
STARTUP_TASKS=essential-seeders:release-2026-07,search-ensure-settings:meili-v1-43-0,search-reindex:release-2026-07
ALLOW_DESTRUCTIVE_STARTUP_TASKS=false
```

Change a version key only when the task must run again.

## Performance screenshot dataset

The <code>performance</code> profile seeds 10,000 assets and 500 people. <code>fresh-db-seed</code> requires <code>ALLOW_DESTRUCTIVE_STARTUP_TASKS=true</code> and is restricted to disposable local or test databases.

## Verification

Review the startup-task ledger and logs. A completed key must skip on restart. A non-blocking reindex may finish after liveness becomes ready.
