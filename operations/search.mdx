Veviad searches assets, people, licenses, and accessories through four prefixed indexes.

## Ensure settings

Run:

```bash
php artisan search:ensure-settings --strict --wait
```

Index creation is idempotent. An expected “already exists” Meilisearch task is accepted, then filterable, sortable, and searchable settings are updated.

## Reindex

Run:

```bash
php artisan search:reindex --strict --wait
```

The command ensures each index, applies settings, clears it, and imports current records. In startup tasks, <code>search-reindex</code> is deliberately non-blocking so a large import cannot hold application readiness.

## When to run

- after first installation or a fresh dataset
- after search schema/settings changes
- after changing the index prefix
- when database and index counts diverge

## Verify

1. Confirm Meilisearch health.
2. Confirm expected document counts for all four indexes.
3. Search a known asset tag, person, license, and accessory.
4. Verify the result opens the correct permitted route.
5. Check task failures and application logs.

## Troubleshooting

A missing index should be created automatically. Repeated failures usually indicate connectivity, API key, prefix, resource, or incompatible Meilisearch-version problems.
