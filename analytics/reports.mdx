## Purpose

Reports turn Veviad data into repeatable operational views. A report definition stores its domain, selected fields, filters, sorting, chart configuration, AI-summary configuration, visibility, schedule, and recipients.

![Machinery report definitions and recent runs](/images/screenshots/reports.webp)

## Before you begin

You need Report view permission; managing, running, scheduling, and administering reports have separate permissions. AI summaries also require an active Veviad AI provider.

## Build a report

1. Open **Reports** and choose **Create report**.
2. Select the domain: assets, people, licenses, or accessories.
3. Choose only the fields needed by readers.
4. Add filters and sorting.
5. Add up to two configurable charts with grouping, metric, limit, and ordering.
6. Enable an AI summary only when a concise interpretation is useful; set locale and optional prompt.
7. Choose visibility.
8. Optionally configure a timezone-aware schedule and recipients.
9. Save and run the report.

![Report builder with chart configuration](/images/screenshots/report-builder.webp)

## Review a run

Open the run to inspect status, row count, chart snapshot, AI-summary status, output, and delivery state.

![Completed report chart and AI summary](/images/screenshots/report-result.webp)

## Expected result

Each run is an auditable snapshot using the execution user's permitted scope. Charts and AI summaries remain attached to that run even if the definition changes later.

## Best practices

- Keep selected fields minimal and filters explicit.
- Use charts for a small number of decisions, not decoration.
- Treat AI summaries as grounded assistance; validate against rows and chart values.
- Use a schedule timezone that matches the business process.
- Avoid sending sensitive exports to broad recipient lists.

## Troubleshooting

A run can complete while its AI summary fails. Check run status, AI-summary status, provider health, permission scope, and recipient delivery separately.
