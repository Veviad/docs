## Purpose

Audit logs answer who changed what, when, and on which record. They support investigation and accountability but do not replace source data or infrastructure logs.

![Audit log listing for machinery operations](/images/screenshots/audit-logs.webp)

## Before you begin

You need Audit Log permission. Some events link to a source record only when you also have permission for that record.

## Investigate a change

1. Open **Audit Logs**.
2. Filter by time, user, action, model, or record.
3. Open the event.
4. Compare old and new values.
5. Follow the source link when available.
6. Correlate the timestamp with Action, report run, handover, or operator logs when needed.

## Expected result

You can identify the actor, affected record, event type, timestamp, and captured field changes.

## Best practices

- Start with a narrow time window.
- Preserve audit history by deactivating referenced configuration instead of deleting it.
- Use audit data for evidence; use application logs for stack traces and service failures.
- Redact personal or secret values before sharing findings.

## Troubleshooting

A missing source link normally means the record was removed or your role cannot view it. If an expected event is absent, confirm the operation actually committed.
