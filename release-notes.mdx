# Track release notes separately from runbooks.

Release Notes summarize noteworthy documentation and platform changes for maintainers. They are intentionally separate from the release-management runbooks, which describe how maintainers plan, ship, and recover releases.

## Highlights

- Current documentation baseline and release-snapshot context
- User-visible documentation or platform changes worth calling out
- A lightweight overview that does not replace tagged GitHub releases or internal release evidence
