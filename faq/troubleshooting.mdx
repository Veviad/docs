Use this page to separate permission, data, browser, and service problems before contacting support.

## Quick checks

1. Reload the page once and confirm you are in the intended workspace.
2. Check whether the feature is permission-dependent.
3. Repeat the action with one known record and note the time.
4. Check the in-app result or error reference; do not copy secrets.
5. Ask an administrator to confirm configuration and service health.

## Common symptoms

| Symptom | Likely cause | Next step |
| --- | --- | --- |
| Menu item is missing | Permission not granted | Ask an administrator to review your role |
| Search misses a recent record | Indexing delay or disabled search | Use direct navigation, then ask an operator to check search |
| Action did not run | Inactive rule, trigger mismatch, or queue issue | Review the Action and queue health |
| Import cannot create an asset | Missing required field or category rule | Correct the reviewed values before confirming |
| Profile field is disabled | SSO or SCIM authority | Change it in the identity provider |

## Contact support

Provide the workspace, route, timestamp, steps, expected result, actual result, and any safe error reference. Redact credentials, tokens, license keys, personal data, and customer records.
