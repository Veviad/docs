## Purpose

Build shared organization data before importing assets so assignments, reporting, automation, and permissions use stable values.

## Before you begin

You need the relevant Settings and Admin permissions. Decide who owns asset data, application access, identity integration, and workflow design.

## Setup order

1. In **Settings → General**, choose currency and shared display defaults.
2. Create locations, departments, cost centers, suppliers, and manufacturers.
3. Configure asset statuses, categories, depreciation rules, tag generation, and custom fields.
4. Configure maintenance types, checklist templates, handover defaults, and notifications.
5. Create or synchronize People.
6. Grant application access to the small subset that needs to sign in; assign roles.
7. Configure integrations, SSO, and the appropriate SCIM purposes.
8. Import a small verified asset sample.
9. Validate global search, saved views, maintenance, handover, Actions, and Reports.
10. Import the remaining inventory.

## Expected result

Records share consistent organizational values, sign-in access is separate from the workforce directory, and administrators can audit every important change.

## Best practices

- Pilot with representative records and permissions before bulk import.
- Keep naming rules short and stable.
- Treat identity and automation changes as controlled rollouts.
- Record the owner of every integration.

## Troubleshooting

If imports reject rows, review required fields and category-specific custom fields. If users cannot sign in, distinguish the Person record from application access.
