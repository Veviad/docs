Veviad is organized around a persistent left-hand navigation, a contextual header, and module-specific views. The main navigation groups align with how teams usually work: operations, analytics, organization, and administration.

![The Veviad sign-in screen with email and password fields](/images/screenshots/sign-in.webp)

## Main navigation

The application sidebar includes the core modules most teams visit every day:

- `Dashboard` for KPIs and operational visibility
- `Assets` for asset records and lifecycle work
- `Licenses` and `Accessories` for supporting inventory
- `People` for user and assignee context
- `Reports` for saved and scheduled reporting
- `Audit Logs` for history and accountability
- `Settings` and `Admin` for configuration and governance

## Header actions

The top navigation area provides quick utilities that stay available while you work:

- global search
- notifications
- help links for system information, documentation, and support
- profile and logout controls

## Global search

Veviad includes a global search experience designed for quick lookup across the workspace.

- Use `Cmd+K` on macOS or `Ctrl+K` on Windows/Linux to open search
- Start typing at least two characters to return results
- Search is debounced to keep the interface responsive
- Results are limited so you can jump quickly instead of browsing a huge list

Use global search when you know roughly what you need and want the fastest possible route to it.

## In-app help

The in-app help menu links back to:

- `docs.veviad.com` for the public manual
- `support.veviad.com` for support workflows
- system information for environment and health visibility

## Good first habits

- Pin your most common navigation path mentally: dashboard, assets, people, reports, or settings
- Use global search for direct lookup and navigation menus for discovery
- Open this docs site in another tab during onboarding so your team has reference material close by
