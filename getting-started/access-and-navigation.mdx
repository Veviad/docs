## Purpose

Use Veviad's consolidated navigation for modules and settings; use global search when you already know the record.

![Veviad sign-in screen](/images/screenshots/sign-in.webp)

## Before you begin

You need an active application user. Local accounts use an invitation or password; SSO-managed accounts use the configured identity provider.

## Sign in and navigate

1. Open your organization's Veviad URL.
2. Sign in locally or choose the configured SSO option.
3. Use the main navigation for **Dashboard, Assets, People,** and **Reports**.
4. Use the organization section for **Audit Logs, Settings,** and **Admin**.
5. On a small screen, open the navigation drawer and select the same destinations.

![Veviad mobile navigation drawer](/images/screenshots/mobile-navigation.webp)

## Find a record

1. Press **Cmd\+K** on macOS or **Ctrl\+K** on Windows/Linux, or open Search.
2. Enter an asset tag, serial number, asset name, or person.
3. Refine by result type when needed.
4. Open the result directly.

## Expected result

Navigation shows only permitted areas. Global search returns current indexed records and opens the canonical detail page.

## Best practices

- Search by asset tag or serial for the most precise match.
- Use saved asset views for recurring lists; use global search for one known record.
- Do not share direct links with users who lack permission.

## Troubleshooting

If sign-in fails, check invitation/access status and identity-provider authority. If search misses a recent item, open its module directly and ask an operator to verify indexing.
