Grenzen Sie Berechtigungs-, Daten-, Browser- und Dienstprobleme ein, bevor Sie den Support kontaktieren.

## Schnellprüfung

1. Laden Sie die Seite einmal neu und prüfen Sie den Arbeitsbereich.
2. Prüfen Sie, ob die Funktion von einer Berechtigung abhängt.
3. Wiederholen Sie den Vorgang mit einem bekannten Datensatz und notieren Sie die Uhrzeit.
4. Notieren Sie das Ergebnis oder eine sichere Fehlerreferenz ohne vertrauliche Zugangsdaten.
5. Lassen Sie Konfiguration und Dienstzustand durch einen Administrator prüfen.

## Häufige Symptome

| Symptom | Wahrscheinliche Ursache | Nächster Schritt |
| --- | --- | --- |
| Menüpunkt fehlt | Berechtigung fehlt | Rolle prüfen lassen |
| Suche findet neuen Datensatz nicht | Indexierungsverzögerung | Direkte Navigation verwenden und Suchstatus prüfen lassen |
| Action läuft nicht | Inaktive Regel, Auslöser oder Hintergrundverarbeitung | Action und Hintergrundverarbeitung prüfen lassen |
| Import kann Asset nicht anlegen | Pflichtfeld oder Kategorieregel | Vorgeschlagene Daten korrigieren |
| Profilfeld ist deaktiviert | SSO oder SCIM verwaltet das Feld | Wert im Identitätsanbieter ändern |
| SSO kann nicht aktiviert werden | Anbieterkonfiguration ist unvollständig oder verbindungskritische Felder wurden geändert | Pflichtfelder ausfüllen, geänderte Verbindung testen und Konfiguration speichern |

## Support kontaktieren

Nennen Sie Arbeitsbereich, Seite, Zeitpunkt, Schritte, erwartetes und tatsächliches Ergebnis sowie eine sichere Fehlerreferenz. Schwärzen Sie Zugangsdaten, Token, API-Schlüssel, personenbezogene Daten und Kundendaten.
