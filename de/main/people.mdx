Personen (/people) ist das Verzeichnis der Belegschaft. Eine Person kann Assets, Lizenzen, Zubehör und Organisationszuordnungen erhalten, ohne ein Anwendungskonto zu besitzen.

Der Anwendungszugriff ist eine separate Berechtigung. Das Erstellen oder Importieren einer Person weist keine Rolle zu, erzeugt keine Anmeldedaten, versendet keine Einladung und erlaubt keine Anmeldung.

## Personen und Anwendungsbenutzer

| Datensatz | Zweck | Anmeldung möglich? |
|---|---|---|
| Person | Identität der Belegschaft für Zuweisungen und Historie | Nur bei aktivem Anwendungszugriff |
| Anwendungsbenutzer | Person mit ausstehendem, aktivem oder gesperrtem Anwendungszugriff | Abhängig von Personen- und Zugriffsstatus |

Verwenden Sie **Personen** für Datensätze der Belegschaft. Verwenden Sie **Admin → Benutzer** für Anwendungszugriff, Anmeldeverfahren, Rollen und Einladungen.

## Verzeichnisansichten

Das Untermenü Personen bietet folgende Standardansichten:

- **Alle Personen** — alle für Sie sichtbaren Personen
- **Aktiv** — Personen, die für neue Zuweisungen verfügbar sind
- **Inaktiv** — aufbewahrte Datensätze, die nicht mehr neu zugewiesen werden können
- **Manager** — als Führungskräfte konfigurierte Personen

Der Name in der Spalte **Person** öffnet das schreibgeschützte Profil. Abhängig von Ihren Berechtigungen können Sie über die Zeilenaktionen das Profil anzeigen oder den administrativen Editor öffnen.

## Personen hinzufügen und importieren

Wählen Sie **Person hinzufügen** für einen einzelnen Datensatz oder **Personen importieren** für einen CSV-Import.

Eine neue Person beginnt mit:

- Personenstatus **Aktiv**;
- Anwendungszugriff **Keiner**;
- keiner Rolle;
- keinem Passwort und keiner SSO-Berechtigung;
- keiner Einladungs-E-Mail.

So kann das vollständige Personenverzeichnis importiert werden, auch wenn nur ein kleiner Teil Veviad verwendet.

## Personen anzeigen und bearbeiten

Das Profil unter /people/{id} ist eine schreibgeschützte Verzeichnisansicht.

Administratoren und delegierte Rollen mit der Berechtigung user.edit können unter /admin/people/{id} den administrativen Editor öffnen. Er enthält Profil- und Organisationsfelder sowie verknüpfte Assets, Lizenzen, Historie und Beziehungsanalyse.

Wenn kein Anwendungszugriff besteht, zeigt der Editor **Kein Anwendungszugriff** und die Aktion **Zugriff gewähren**. Der Tab Rollen ist nicht verfügbar, weil Rollen ausschließlich die Berechtigungen angemeldeter Benutzer steuern.

![Der administrative Personen-Editor für einen Datensatz ohne Anwendungszugriff](/images/screenshots/de/admin-person-no-access.webp)

## Personen-Lebenszyklus

Berechtigte Administratoren ändern den Aktiv-Schalter im administrativen Editor.

Das Deaktivieren einer Person:

- erhält bestehende Zuweisungen, Verwahrungsdaten und Audit-Historie;
- entfernt die Person aus neuen Zuweisungslisten;
- lässt die unabhängige Zugriffsberechtigung unverändert;
- blockiert die Anmeldung sofort und widerruft aktive Sitzungen und API-Token, wenn Zugriff vorhanden ist.

Die Reaktivierung stellt nur den Personenstatus wieder her. Sie gewährt keinen Zugriff und ändert keine Rolle.

Bei über Verzeichnis-SCIM verwalteten Personen sind Profil- und Lebenszyklusfelder in Veviad schreibgeschützt. Nehmen Sie Änderungen im Identitätsanbieter vor.

## Welcher Bereich ist der richtige?

| Aufgabe | Bereich |
|---|---|
| Personen der Belegschaft hinzufügen oder importieren | Personen |
| Zuweisungen und Historie anzeigen | Personen |
| Profil- oder Organisationsdaten bearbeiten | Admin → Personen |
| Person aktivieren oder deaktivieren | Admin → Personen |
| Anwendungszugriff gewähren, sperren oder wiederherstellen | Admin → Benutzer oder administrativer Personen-Editor |
| Lokale Einladung versenden oder verwalten | Admin → Benutzer / Einladungen |
| Anwendungsrolle zuweisen | Admin → Personen, nachdem Zugriff besteht |
| Vom Identitätsanbieter verwalteten Datensatz ändern | Identitätsanbieter |
