## Zweck

Personen ist das Verzeichnis für Zuweisungen, Verwahrung, Abteilungen, Standorte und Berichte. Eine Person benötigt keinen Anwendungszugriff.

![Personenverzeichnis im Maschinenbau](/images/screenshots/de/people.webp)

## Voraussetzungen

Sie benötigen Personen-Anzeigerechte. Erstellen, Bearbeiten und Anwendungszugriff besitzen getrennte Berechtigungen.

## Person verwalten

1. Öffnen Sie **Personen** und suchen Sie nach Name, E-Mail, Abteilung, Standort oder Status.
2. Öffnen Sie den Datensatz.
3. Pflegen Sie lokal autoritative Identitäts- und Organisationsfelder.
4. Prüfen Sie zugewiesene Assets und Historie.
5. Lassen Sie den Anwendungszugriff unverändert, wenn keine Anmeldung nötig ist.

## Anwendungszugriff erteilen

1. Öffnen Sie **Admin → Benutzer**.
2. Wählen Sie die bestehende Person.
3. Erteilen Sie Zugriff, Rolle und gegebenenfalls Einladung.
4. Entziehen Sie den Zugriff später, ohne die Person zu löschen.

![Anwendungszugriff erteilen](/images/screenshots/de/admin-grant-access.webp)

## Erwartetes Ergebnis

Das Personenverzeichnis kann die gesamte Belegschaft enthalten; Admin → Benutzer nur anmeldeberechtigte Personen.

## Bewährte Vorgehensweisen

- Legen Sie eine Person pro Identität an.
- Erzeugen Sie keinen zweiten Benutzer für Zugriffsprobleme.
- Lassen Sie SCIM für synchronisierte Felder autoritativ.
- Bewahren Sie Historie inaktiver Personen, entziehen Sie aber Zugriff.

## Fehlerbehebung

Eine in Admin fehlende Person besitzt normalerweise keinen Anwendungszugriff. Deaktivierte Felder können IdP-verwaltet sein.
