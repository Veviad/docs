## Zweck

Verwenden Sie die konsolidierte Navigation für Module und Einstellungen; nutzen Sie die globale Suche für einen bereits bekannten Datensatz.

![Veviad-Anmeldung](/images/screenshots/de/sign-in.webp)

## Voraussetzungen

Sie benötigen einen aktiven Anwendungsbenutzer. Lokale Konten verwenden Einladung oder Passwort; SSO-Konten den Identitätsanbieter.

## Anmelden und navigieren

1. Öffnen Sie die Veviad-URL Ihrer Organisation.
2. Melden Sie sich lokal oder per SSO an.
3. Nutzen Sie die Hauptnavigation für **Dashboard, Assets, Personen** und **Berichte**.
4. Nutzen Sie den Organisationsbereich für **Einstellungen, Zugriffsverwaltung und das Audit-Protokoll**.
5. Öffnen Sie auf kleinen Bildschirmen den Navigations-Drawer.

![Mobile Veviad-Navigation](/images/screenshots/de/mobile-navigation.webp)

## Datensatz finden

1. Drücken Sie **Cmd\+K** unter macOS oder **Ctrl\+K** unter Windows/Linux.
2. Suchen Sie nach Asset-Tag, Seriennummer, Asset-Name oder Person.
3. Grenzen Sie den Ergebnistyp ein.
4. Öffnen Sie den Treffer.

## Erwartetes Ergebnis

Die Navigation zeigt nur erlaubte Bereiche. Die Suche öffnet den kanonischen Detaildatensatz.

## Bewährte Vorgehensweisen

- Suchen Sie möglichst nach Asset-Tag oder Seriennummer.
- Verwenden Sie gespeicherte Asset-Ansichten für wiederkehrende Listen.
- Teilen Sie keine Direktlinks mit Personen ohne Berechtigung.

## Fehlerbehebung

Prüfen Sie bei Anmeldeproblemen Einladung, Zugriffsstatus und IdP-Autorität. Fehlt ein Suchtreffer, öffnen Sie das Modul direkt und lassen Sie die Indexierung prüfen.
