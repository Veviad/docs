A hotfix resolves an urgent production problem with minimal scope and a proven rollback point.

## Before you begin

Confirm the affected version, source SHA, image digest, reproduction, and business impact. Capture a current backup before any data change.

## Procedure

1. Branch from the affected stable line or exact release tag.
2. Implement only the cause, tests, and required documentation.
3. Run PHP tests, frontend tests, build, and quality checks.
4. Open the hotfix PR against the approved stable base.
5. Create and verify the new patch tag and image digest.
6. Deploy the digest; check <code>/health</code>, sign-in, and the affected workflow.
7. Carry the fix back into <code>develop</code>.

## Stop conditions

Stop for an unclear migration, missing backup, unreproduced impact, or mismatched digest.
