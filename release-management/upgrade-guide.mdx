Upgrade Veviad with a verified release digest and a documented rollback point.

## Before the upgrade

- read release notes and migrations
- back up PostgreSQL and persistent storage
- verify capacity, PHP 8.4, and dependencies
- record target digest and current version
- define maintenance and rollback windows

## Procedure

1. Update the deployment to the approved digest.
2. Start PostgreSQL, Redis, and Meilisearch before the application.
3. Allow Laravel migrations and idempotent startup tasks to complete.
4. Ensure search settings; a large reindex may continue as a non-blocking startup task.
5. Check <code>/up</code>, then <code>/health?verbose=true</code> where policy permits.
6. Test sign-in, Dashboard, Assets, search, queue processing, and a file upload.
7. Confirm version, SHA, and digest in System Information.

## After the upgrade

Watch error logs, queue age, search indexing, and the storage write probe. Roll back only according to the pre-approved plan.
