This handbook defines the releasable path from <code>develop</code> to a verified production artifact.

## Runtime baseline

Current releases use Laravel 13 on PHP 8.4. <code>develop</code> is the integration line for upcoming work; stable tags and image digests are immutable release evidence.

## Release flow

1. Synchronize <code>develop</code> and confirm every intended PR is included.
2. Derive the version only through the repository's version scripts and <code>BASE_VERSION</code>.
3. Run tests, static analysis, frontend build, and security checks.
4. Open the release PR according to the current workflow.
5. After merge, verify the tag, GitHub release, container tag, digest, and attestation.
6. Exercise the upgrade and health checks in a representative environment.
7. Record source SHA, version, digest, migrations, and rollback decision.

## Required evidence

- source SHA and base branch
- test and quality checks
- immutable image digest
- migration and backup status
- <code>/health</code> payload and critical smoke tests

## Rules

Never move a published tag. Promote the same tested digest between environments. A hotfix contains only the urgent correction and its required tests.
