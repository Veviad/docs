Veviad separates the repository's product base version from build, prerelease, and release labels.

## Contract

- <code>BASE_VERSION</code> is the authoritative product base.
- Repository scripts derive the published version.
- Git tags and container images reference the same source SHA.
- The image digest is the immutable runtime identity.

## Procedure

1. Read <code>BASE_VERSION</code> from the target SHA.
2. Use <code>scripts/generate-version.sh</code> or the release workflow.
3. Check SemVer format and existing tags.
4. Do not publish a manually calculated version.
5. Record tag and digest with the release evidence.

## Patch rule

A stable patch starts from the approved stable line and excludes future <code>develop</code> changes. After release, carry the correction back into <code>develop</code>.

## Troubleshooting

If the UI version, Git tag, and image metadata disagree, stop promotion. Resolve the source SHA and digest instead of overwriting labels.
