# Settings

Settings (`/settings`) is a tile grid of shared configuration areas. Each tile opens a dedicated page. Use Settings for shared structure and system-wide rules — not for one-off record edits.

![The settings tile grid with shared configuration areas](/images/screenshots/settings.webp)

_View shown: `demo.veviad.cloud/settings`_
## Settings areas

| Tile | What it configures |
|---|---|
| **General** | Organization currency and display settings |
| **Cost Center** | Cost centers and budget allocations |
| **Depreciations** | Depreciation schedules applied to assets |
| **Departments** | Org departments (used by People, Assets, and reports) |
| **Locations** | Physical sites and locations |
| **Status Labels** | Asset status labels and types (drives Active / In Storage / Deprecated / Disposed views) |
| **Categories** | Asset categories (drives the Dashboard "Categories" KPI) |
| **Maintenance Types** | Type options used when adding asset maintenance records |
| **Suppliers & Manufacturers** | Vendor directory for purchase and warranty records |
| **Asset Tag Configuration** | How new asset tags are generated |
| **Asset Tag Label** | Printable QR code layout for physical labels |
| **Field Visibility & Required Fields** | Which record fields are visible and which are mandatory |
| **Handover** | Acknowledgement mode, external links, and PDF access for asset handovers |
| **Actions** | Event-driven and scheduled automation triggers (see Workflows) |
| **Checklists** | Reusable checklist templates (see Workflows) |
| **Notifications** | Notification settings and alert thresholds |

## Rules of thumb

- If it shapes how **many** records behave → it belongs here.
- If it only changes **one** record → it belongs on that record.

## Recommended setup order for a new workspace

1. **General** — set organization currency
2. **Departments** and **Locations** — the taxonomy People and Assets reference
3. **Categories**, **Status Labels**, and **Maintenance Types** — the taxonomy Assets and maintenance records reference
4. **Suppliers & Manufacturers** — referenced from purchase info
5. **Asset Tag Configuration** and **Asset Tag Label** — before bulk import
6. **Depreciations** — before reporting financial KPIs
7. **Notifications**, **Actions**, **Checklists** — once operational data is flowing

Getting this order right reduces cleanup later: assets imported before categories exist will need backfilling.

## Maintenance type rules

Maintenance types are global options for asset maintenance records. Active types are available for new maintenance entries and scheduled maintenance actions. If a type is already used by records, deactivate it instead of deleting it so historical service data remains readable.
