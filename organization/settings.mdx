## Choose the right area

| Responsibility | Configuration |
| --- | --- |
| Shared organization data | General, locations, departments, cost centers, suppliers, manufacturers |
| Asset lifecycle | Statuses, categories, depreciation, tag generation, labels, custom fields, field rules |
| Work execution | Checklist templates, maintenance types, Actions, notification defaults, handover defaults |
| User preferences | Profile language, timezone, and notification preferences |
| Sign-in and provisioning | Admin users, invitations, roles, integrations, SSO, and SCIM |

## Permission behavior

Settings uses granular permissions. A user can manage one configuration area without seeing all Settings pages.

## Best practices

- Configure shared values before importing records.
- Deactivate values that are already referenced.
- Test changes with one representative record.
- Review Audit Logs after material configuration changes.
