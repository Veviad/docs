## Purpose

Admin owns application access and platform integrations. It does not replace the People directory.

## Main areas

- **Users**: people with application access or a pending access lifecycle.
- **Invitations**: local activation, expiry, resend, and revocation.
- **Roles**: permission bundles assigned to application users.
- **Integrations**: AI, OCR, email, identity, and provisioning configuration.
- **Login content and links**: organization-specific sign-in presentation where enabled.
- **System Information**: version and runtime health for administrators.

## Best practices

- Grant access to an existing person.
- Use least privilege and test roles with non-admin accounts.
- Revoke access instead of deleting workforce history.
- Keep integration secrets in the dedicated protected fields.
