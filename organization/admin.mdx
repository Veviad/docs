The Admin area (`Organization → Admin`) is where authentication, access, and org-level integrations are configured. It has five subpages: **General**, **Roles**, **Users**, **Invitations**, and **Integrations**.

![The Admin Dashboard with users, assets, license seats, and system health](/images/screenshots/admin.webp)

_View shown: `demo.veviad.cloud/admin`_

## General (Admin Dashboard)

Landing page for the admin area. At a glance:

- **KPI tiles:** Users, Assets, License Seats, Departments — each with totals and a percentage bar.
- **Role Distribution** — seat count and share per system role (Admin, Asset Manager, Department Manager, User).
- **Recent Activities** — latest admin-relevant actions.
- **System Health** — Permissions count, Roles count, Database Size, Storage Usage.
- **Integrations status** — Single-Sign-On and Veviad AI with active / inactive badges.

## Roles

![The Roles overview with permissions per role](/images/screenshots/admin-roles.webp)

_View shown: `demo.veviad.cloud/admin/roles`_

Manage the permission bundles that users can be assigned. The default role set ships with:

- **Admin** — full platform access
- **Asset Manager** — lifecycle ownership of assets
- **Department Manager** — scoped access within a department
- **User** — baseline read / use access

Create custom roles when the built-ins don't fit. The workspace tracks the total permission count (≈79 in the default seed) and the number of active roles under System Health.

## Users

![The Users list with roles and account status](/images/screenshots/admin-users.webp)

_View shown: `demo.veviad.cloud/admin/users`_

The admin view of the user directory — login accounts and their assigned roles. This is different from `People` (which is the HR / asset-assignment view): Users is the authentication-and-permission lens, People is the operational lens.

## Invitations

![The Invitations view with pending invitations](/images/screenshots/admin-invitations.webp)

_View shown: `demo.veviad.cloud/admin/invitations`_

Controlled onboarding for new users.

**Administrator flow:**

1. open `Admin → Users` and choose `Invite User`
2. enter the recipient email
3. assign one or more roles
4. send the invitation

**Invitation lifecycle:**

- pending state until accepted
- expiry window (commonly 72 hours, configurable)
- resend support if the email is lost or expires
- revoke support when access should be cancelled before acceptance

**Invitee flow:** the user receives an email with a secure acceptance link and creates their password via the acceptance page.

**If an invitation doesn't arrive:** check spam, verify the email address, resend, and confirm mail delivery with your administrator if the problem continues.

## Integrations

![The Integrations list with Single-Sign-On and Veviad AI](/images/screenshots/admin-integrations.webp)

_View shown: `demo.veviad.cloud/admin/integrations`_

Two integrations ship by default and are toggled here:

- **Single-Sign-On** — inactive by default; configure via the per-row gear icon.
- **Veviad AI (Chat Assistant)** — inactive until enabled and configured; use the per-row gear icon to configure the AI provider, then enable the integration for Assistant and PDF extraction workflows.

Enable SSO before you invite users in larger rollouts — reassigning authentication after invitations are accepted adds operational friction.

For Microsoft Entra ID setup, SCIM provisioning, and group-to-role synchronization, see [SSO and SCIM](/organization/sso-scim).

## System visibility

The Admin Dashboard is where you look when you need a one-screen answer to "is this tenant healthy?": users total vs. active, assets total vs. utilized, license seats, recent activity, and integration state. For event-level detail, go to `Analytics → Audit Logs`.
