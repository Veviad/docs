Assets are the core record type. Each asset carries its identity (tag, serial, model), its assignment state, purchase and warranty context, depreciation, and a full history trail.

![The assets list with default views, asset tags, names, and assignments](/images/screenshots/assets.webp)

## The assets list (`/assets`)

A second-level pane on the left organizes filtered views:

**Default views**

- **All** — every asset in the workspace
- **Active** — deployed assets currently in use
- **Unassigned** — assets not yet handed out
- **In Storage** — warehouse / spare stock
- **Deprecated** — flagged as end-of-life but still on record
- **Disposed** — written off
- **Archived** — retired, kept for history

**Custom views** — save your own filter combinations for reuse.

## Toolbar actions

From the table header (top-right):

| Icon | Action |
| --- | --- |
| `+` | Add a new asset |
| ✨ | AI-assisted actions (e.g. Veviad AI Import) |
| Bookmark | Save the current filter as a custom view |
| Trash | Bulk delete selected rows |
| Download | Export the current view |
| Upload | Import assets from file |

Per-column filter funnels let you narrow by ID, Tag, Name, Assignee, Department, or Category.

## Asset detail panel

Clicking a row opens a seven-tab panel:

- **Details** — General Information (Asset Name, Tag, Serial, Model, Manufacturer, Category, Status), Assignment (Assigned To, Location, Department, Last Checkout, Expected Checkin), Purchase Information (Date, Cost, Order Number, Invoice Number, Supplier), Warranty Information (Expires, Period, Status badge), and Timestamps (Created, Created By, Last Updated, Last Updated By).
- **Checklists** — per-asset compliance templates and their status.
- **History** — time-ordered audit of actions against this asset.
- **Handover** — receipt and return confirmations, PDF handover records, and pending follow-up for assigned or returned equipment.
- **Maintenance** — open and completed maintenance records, scheduled dates, optional notes, cost and warranty extension details, plus PDF report / invoice attachments.
- **Visual Analyzer** — AI-assisted visual inspection for the asset.
- **Asset Tag Label** — printable QR code label for the physical asset.

## Common tasks

- **Add an asset** — `+` toolbar action or `✨` for PDF-based bulk intake
- **Assign / unassign** — from the asset detail or the row menu
- **Confirm handover** — use the Handover tab to review employee receipt and return records, including PDF previews and downloads
- **Document maintenance** — open the asset's Maintenance tab to add performed work, schedule upcoming work, or mark records as completed
- **Attach reports / invoices** — upload service reports and invoices directly on saved maintenance records
- **Archive / dispose** — use Status Labels (configured in Settings → Status Labels) and let the default views pick them up
- **Track warranty** — fill Warranty Expires \+ Period; the Warranty Status badge flags expired items

## Practical notes

- the **Category** field drives the dashboard "Categories" KPI — keep categories meaningful and few
- **Asset Tags** are generated according to your tag configuration (`Settings → Asset Tag Configuration`)
- printable QR labels use `Settings → Asset Tag Label` formatting
- maintenance type options are configured globally in `Settings → Maintenance Types`
- everything that happens to an asset is recorded in `Analytics → Audit Logs` with Entity = "Asset"
