## Purpose

Assets are the primary lifecycle records in Veviad. The grid is designed to work with large inventories—such as 10,000 assets—by loading rows as needed instead of rendering the full dataset at once.

![Lazy-loaded machinery asset grid](/images/screenshots/assets.webp)

## Before you begin

You need Asset view permission. Create, edit, delete, import, maintenance, handover, and checklist operations have separate permissions.

## Work with the asset grid

1. Open **Assets**.
2. Search or filter by status, category, assignment, location, department, cost center, or custom field.
3. Sort and arrange columns.
4. Save the current grid state as a personal view.
5. Open an asset to review identity, finance, assignment, custom data, maintenance, handover, checklist, invoice, and audit history.

![Machinery asset detail](/images/screenshots/asset-detail.webp)

## Import assets

1. Select **Import** and download or follow the current template guidance.
2. Map or validate required fields, categories, statuses, and custom fields.
3. Review validation errors before confirming.
4. Import a small sample first, then the remaining batch.
5. Search for representative asset tags after indexing completes.

![Asset import workflow](/images/screenshots/asset-import.webp)

## Expected result

The grid keeps server-side filters and sorting while lazily loading rows. Saved views restore your grid configuration without changing other users.

## Best practices

- Use the asset tag as the stable operational identifier.
- Keep one source of truth for category, location, and ownership.
- Record maintenance and handover on the asset rather than in free-text notes.
- Avoid exporting the whole inventory when a saved view or report answers the question.

## Troubleshooting

If a row count or search result appears stale, reload the view and verify search indexing. If a field is missing, check category-specific custom-field configuration and permission.
