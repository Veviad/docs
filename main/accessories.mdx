# Accessories

Accessories are for items that support the main asset estate but do not behave like primary assets — cables, chargers, monitors, docks, adapters, and similar. Keeping them separate from the asset register prevents bloat while still giving teams visibility.

![The accessories list with stock and assignments](/images/screenshots/accessories.webp)

_View shown: `demo.veviad.cloud/accessories`_
## What the accessories list shows

Filtered views available at `Main → Accessories`:

- **All** — every accessory in the workspace
- **Active** — currently in use
- **In Storage** — unassigned stock
- **Archived** — retired / written off

**Custom views:**

- **Unassigned Accessories** — stock available to hand out
- **My Accessories** — accessories assigned to the current user

## Record fields

Each accessory row exposes:

- **ID** — internal identifier
- **Serial** — serial number where applicable
- **Name** — descriptive item name
- **Category** — taxonomy group
- **Status** — active, in storage, or archived

## Common tasks

- add an accessory (`+` toolbar action)
- assign to a person or a location
- move to "In Storage" when it returns to stock
- archive when an accessory is retired

## Accessories vs. assets

- use **Assets** for items tracked individually with lifecycle, warranty, depreciation (laptops, workstations, phones, network equipment)
- use **Accessories** for peripheral items tracked in simpler terms (cables, adapters, mice)
- use **Licenses** for software entitlements

Put items in the right type early — moving a record across types later is not a one-click operation.
