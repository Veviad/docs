## Purpose

The dashboard is a personal workspace, not a fixed company scorecard. Arrange standard and configurable widgets around the decisions you make most often.

![Machinery dashboard with configurable widgets](/images/screenshots/dashboard.webp)

## Before you begin

You need Dashboard access. Creating configurable analytics widgets also requires access to the data domain used by the widget.

## Customize the dashboard

1. Open **Dashboard**.
2. Select **Edit dashboard**.
3. Add a standard widget or open the widget builder.
4. Choose the data domain, visualization, metric, grouping, filters, and title.
5. Preview the widget before saving.
6. Drag widgets into order and resize supported cards.
7. Save the layout.

![Dashboard edit mode](/images/screenshots/dashboard-edit.webp)

![Configurable widget builder](/images/screenshots/dashboard-widget-builder.webp)

## Expected result

The saved layout is personal to your user. Widgets load their data independently, and the remaining dashboard stays usable if one widget cannot load.

## Best practices

- Keep the first row limited to the measures that change daily decisions.
- Use a report for detailed row-level analysis; use the dashboard for signals.
- Name custom widgets by question, such as “Machines due for service”.
- Reset to the default layout when testing or recovering from an unusable arrangement.

## Troubleshooting

A blank widget can mean no matching data, a removed field, or missing domain permission. Edit the widget and preview its configuration.
