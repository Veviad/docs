# Maintenance

Maintenance keeps service history close to the asset it belongs to. Open an asset and go to `Maintenance` to document completed work, schedule upcoming work, or attach the PDF evidence that supports the record.

## What the tab shows

The Maintenance tab starts with a compact summary:

- **Open** — maintenance records that are not completed yet
- **Overdue** — open records with a scheduled / performed date before today
- **Last completed** — most recent completed maintenance date
- **Next scheduled** — next open maintenance date

Open and completed records are separated so teams can quickly see what still needs attention without losing the asset's service history.

## Add a maintenance record

Use **Add Maintenance** from the asset detail panel.

Each record captures the essential operational context:

- **Title** — the short name of the work, such as "Quarterly inspection"
- **Type** — a configured maintenance type, such as Preventive, Repair, Inspection, Calibration, or Other
- **Scheduled / performed date** — the planned date for open work, or the date the work happened
- **Completed** — marks whether the work is still open or already finished
- **Completion date** — the actual finish date when the record is completed
- **Notes** — optional plain-text details, findings, or follow-up instructions
- **Cost** — optional cost for reporting and later review
- **Warranty Extension (months)** — optional warranty extension gained through the work

Supplier is not part of the visible maintenance workflow. Keep supplier and manufacturer information on the asset's purchase details unless your internal process requires a separate note.

## Attach reports and invoices

Saved maintenance records can hold two PDF attachments:

- **Report** — service report, inspection result, calibration certificate, or completion evidence
- **Invoice** — supplier invoice or billing document for the work

Each attachment area supports **Upload**, **View**, **Download**, **Replace**, and **Remove**. Attachments are added after the maintenance record exists, which keeps draft records simple and avoids orphan uploads.

## Scheduled maintenance from Actions

Maintenance can also be created automatically from `Settings → Actions`.

Use the **Create Scheduled Maintenance** action when the same work should be created for matching assets:

- after an asset is created
- after an asset status changes
- on a reusable schedule, such as monthly or yearly

Scheduled actions use the current asset filters at run time. For example, a monthly inspection action can target all active assets in one category and create an open maintenance record for each matching asset.

## Configure maintenance types

![The Maintenance Types settings page](/images/screenshots/settings-maintenance-types.webp)

_View shown: `demo.veviad.cloud/settings/maintenance-types`_

Admins manage the available type list in `Settings → Maintenance Types`.

Active types are available on new records. Existing records keep their stored type label, so old service history stays readable even if a type is later renamed or deactivated. If a type has already been used, deactivate it instead of deleting it.
