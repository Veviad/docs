## Purpose

People is the workforce directory used for asset and license assignments, custody, departments, locations, and reporting. A person does not need application access.

![People directory with machinery workforce records](/images/screenshots/people.webp)

## Before you begin

You need People view permission. Creating or editing people and managing application access require separate permissions.

## Manage a person

1. Open **People** and search by name, email, department, location, or status.
2. Open the person record.
3. Maintain locally authoritative identity and organization fields.
4. Review assigned assets and related history.
5. Leave application access unchanged unless the person must sign in.

## Grant application access

1. Open **Admin → Users**.
2. Select the existing person.
3. Grant access, choose a role, and send an activation invitation when required.
4. Revoke access later without deleting the person.

![Grant application access to an existing person](/images/screenshots/admin-grant-access.webp)

## Expected result

The workforce directory can contain all assignable people while Admin → Users shows only those with sign-in access.

## Best practices

- Create one person per human identity.
- Do not create a second user to solve an access problem.
- Let SCIM remain authoritative for synchronized fields.
- Keep inactive former workers for historical assignments, then remove access.

## Troubleshooting

A person missing from Admin → Users normally has no application access. A disabled field may be identity-provider managed.
