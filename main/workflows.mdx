## Purpose

Actions automate repeatable asset work. They can react to asset events or run on a schedule, then create checklists, maintenance, handover requests, or notifications.

![Actions listing with event-driven and scheduled rules](/images/screenshots/settings-actions.webp)

## Before you begin

You need Action permissions. Referenced templates, types, statuses, and recipients must already exist. Queue workers must be healthy.

## Create an Action

1. Open **Settings → Actions**.
2. Choose an event trigger such as asset creation, status change, or assignment change—or choose **Scheduled**.
3. Set narrow trigger conditions.
4. Choose the outcome and configure its required values.
5. For scheduled Actions, select frequency, local time, and an explicit timezone.
6. Save the Action inactive.
7. Test with a controlled asset, then activate it.

## Expected result

Matching events or due schedules dispatch the configured work and record the resulting activity.

## Best practices

- Prefer several narrow Actions over one broad rule.
- Keep new Actions inactive until a real test succeeds.
- Avoid duplicate triggers that create the same checklist or maintenance.
- Treat schedule timezone as part of the rule's business meaning.

## Troubleshooting

Review active state, trigger conditions, referenced configuration, next run time, queue worker heartbeat, and failed jobs. A saved Action is not proof that it executed.
