# Licenses

Licenses in Veviad are a first-class record type alongside assets and accessories. Use them to track software ownership, subscription validity, and the seat-level allocation that matters for procurement and compliance.

![The licenses list with seat allocation and validity](/images/screenshots/licenses.webp)

_View shown: `demo.veviad.cloud/licenses`_
## What the licenses list shows

The licenses list (`Main → Licenses`) supports filtered views:

- **All** — every license in the workspace
- **Active** — licenses currently in use
- **Expired** — past-end-date licenses still on record
- **Archived** — retired licenses kept for history

**Custom views** (saved filters):

- **Unassigned Licenses** — seats available to assign
- **My Licenses** — licenses assigned to the current user

## Record fields

Each license row exposes:

- **ID** — internal identifier
- **License Key** — the key itself (masked where appropriate)
- **Name** — product or bundle name
- **Type** — software type / category
- **Status** — active, expired, or archived

## Common operational tasks

- add a new license (`+` toolbar action)
- assign or release a seat
- update the expiry date when a renewal lands
- archive a license instead of deleting so reporting history is preserved

## Practical notes

- license utilization is tracked from the Dashboard and from the Admin Dashboard KPI tiles
- license seat math (assigned vs. total) is what drives the dashboard utilization percentages
- expired licenses remain queryable — use Archived when a license is truly retired
