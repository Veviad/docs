The Veviad AI Assistant is the natural-language layer over your workspace. Open it from the floating purple sparkle button (bottom-right of any page) to ask questions in plain language and get structured answers back.

![The Veviad AI Assistant panel with a results table and Copy CSV action](/images/screenshots/assistant.webp)

_View shown: `demo.veviad.cloud`_
## What the assistant can do

When opened, the assistant auto-populates with a live workspace snapshot:

- asset counts by **Category** (Laptops, Workstations, Network Equipment, Mobile Phones, …)
- asset counts by **Status** (In stock, Deployed, …)
- asset counts by **Department**
- combined asset value

Results render as embedded tables with **Copy CSV** on each — not just free-text prose. This makes the assistant useful for fast analysis as well as question-answering.

## Chat interface

- **Provider selector** — switch the underlying AI model. Additional providers can be added through `Organization → Admin → Integrations → Veviad AI`.
- **Export** — save the current conversation.
- **Clear chat** — reset the session.
- **Input** — `Press Enter to send, Shift+Enter for new line`.

## Where it helps most

- find records fast ("show me deployed laptops in the Human Resources department")
- summarize a record set before a review
- surface cross-module context (an asset's assignment, warranty, and recent checklist history in one breath)
- prepare to act — the assistant is a shortcut into the right filter, not a replacement for the record itself

## Best practices

- ask focused questions when possible — the assistant handles multi-part asks, but narrow ones return cleaner tables
- confirm important decisions against the underlying records in `Main → Assets` / `Main → People`
- pair AI results with `Analytics → Reports` and `Analytics → Audit Logs` for governance-sensitive workflows
- treat AI output as a starting point for action, not an authoritative record

## Configuring providers

Go to `Organization → Admin → Integrations` and click the gear icon on the **Veviad AI** row. From there you can enable/disable providers and select which model the assistant uses by default.
