## Purpose

PDF intake reduces manual entry from invoices, delivery notes, and equipment documents. Extraction produces a reviewable candidate, not an automatic source of truth.

![PDF asset intake review](/images/screenshots/pdf-asset-intake.webp)

## Before you begin

You need Asset create and AI-import permission, an active AI/OCR integration, and a supported PDF without secrets unrelated to the asset.

## Extract and create

1. Open the Veviad AI import flow.
2. Upload the PDF.
3. Wait for OCR and extraction to finish.
4. Review every proposed asset and field.
5. Correct category, status, tag, serial, cost, dates, supplier, manufacturer, and custom fields.
6. Resolve required-field validation.
7. Confirm only the reviewed records.
8. Open each created asset and verify attachments and values.

## Expected result

Veviad creates only confirmed assets and retains the extraction job status for troubleshooting.

## Best practices

- Split unrelated documents into separate jobs.
- Redact bank data, personal data, and credentials before upload.
- Never accept a guessed serial or cost without source evidence.
- Use bulk import instead when the source is already structured.

## Troubleshooting

Check file type and size, OCR provider health, job status, queue health, and required fields. A failed extraction does not create an asset.
